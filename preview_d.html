Oversec

The Encryption-Swiss-Army-Knife.

Oversec allows you to transparently encrypt and decrypt any text in any app[3] - use it to exchange encrypted chat- or email messages or store your own encrypted notes on your phone.

Oversec is completely agnostic of the app being used, it works with Whatsapp, Line, Snapchat or any other chat app. It also works with the Gmail app or any other Email app.

End-to-End Encryption was yesterday. Oversec introduces "Eye-to-Eye" encryption. Encrypted data is only decrypted when shown on the screen and only _while_ it is shown! No clear text is ever persisted and thus cannot be extracted from the file system or accidentally be uploaded into the cloud.

We also carefully designed Oversec so that it doesn't require internet access. Because of that, you can rest assured that no decrypted information can ever leave your device.

How it works:

Oversec constantly monitors the text on your screen (via the Accessibility API). When it finds an encrypted text, it tries to decrypt it and then shows the decrypted text as an overlay in place of the encrypted text.

In order to encrypt a text, Oversec shows a button next to any active text-input field. After having entered the clear-text in the input field, pressing this button lets Oversec read the clear-text, encrypt it and put back the encrypted text into the input field. It is now ready to be sent in the underlying app.

Oversec also features a unique way of encoding the encrypted messages. It stores the encrypted text in invisible (zero-width) characters and let's you add decoy text at the end. This way, a message will just e.g. show "How are you today" with no visible sign of any encryption, whereas in reality it contains an encrypted message "Meet at PB Hotel now".

The encoding however can be changed depending on the app: For Email apps e.g. one would use ASCII-armoured PGP encoding - that way such messages can be decrypted with a regular E-mail client supporting PGP encrypted mails. For SMS/Texting, BASE64 encoding can be used to make sure that the message doesn't get corrupted while traveling through the GSM infrastructure.

Oversec encrypts your data either using symmetric keys  (using ChaCha20 cipher + Poly1305 MAC [1]) or using asymmetric PGP encryption (making use of the Open-Keychain app [2]).

The encryption related code will be open-sourced soon (pending some cleanup and documentation.)



[1] Other symmetric ciphers and MACs, especially the ones included in "BouncyCastle" JCE, can be added with minimal effort.

[2] Open-Keychain Homepage

[2] All native apps are supported. Apps that use a WebView Browser component and the Browser app itself are not supported
